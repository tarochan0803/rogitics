# 道幅AIの実現性評価と商用化に向けたライセンス/法務方針

作成: 2026-07-10（L4SIM デバッグセッション）
対象読者: 本プロダクトの商用化判断をする人。結論から書く。

---

## 1. 結論（道幅は何で取るべきか）

「YOLOで詳細な道幅が取れるのか？」への正直な答え:

| 手段 | 精度の期待値 | 商用データ源としての可否 |
|------|------------|----------------------|
| **基盤地図情報 道路縁 (fgd_edge)** | 実測級（全幅） | ◎ 最優先。既に統合済み（confidence 0.88） |
| **航空写真セグメンテーション + 幾何計測 (road_seg)** | 良条件で ±0.5m 程度 | ○ 第二優先。未タグ道路の補完に有効 |
| **Street View + YOLO (streetviewScan)** | ±1〜2m（単眼推定の限界） | △ **保存禁止**（後述の Google ToS 問題）。画面上の参考表示まで |
| **手動上書き (userOverrideWidth)** | 人の判読 | ◎ authoritative（現行どおり） |

- **「YOLOに幅を覚えさせる」設計はそもそも採っていない**（road_seg/README の設計が正しい）。
  航空写真から道路面マスクを推定し、幅は OSM 中心線への垂線×多断面中央値の幾何計算で出す。
  精度の生命線はモデルではなく m/pixel 換算（z18 ≈ 0.49m/px）と多断面の中央値。1断面では
  ±1px≒±0.5m なので、**「詳細な道幅」= ±0.5m 級が実用上限**。それ以上の精度は航空写真の
  分解能上、原理的に出ない。狭隘路の通行可否判定には十分（判定余裕のバケットが 0.4/1.0m 刻み）。
- 樹木・建物影・高架下は断面が汚れる → frameConfidence で落ちる設計は妥当。**fgd_edge が
  ある道路では航空写真推定を使わない**（現行の融合優先度どおり）。
- 教師データ現況: 手動マスク約20枚 + YOLO要素ラベル100枚は PoC 規模。商用品質には
  道路面マスクを数百枚規模へ増やす必要がある（ラベルサーバは整備済み）。

## 2. ライセンス問題（重要・要対応）

### 2.1 Ultralytics YOLO は AGPL-3.0（商用の地雷）
- `road_seg/train_yolo.py` / SV側のYOLO検出で使う **Ultralytics は AGPL-3.0**。
  AGPLは「ネットワーク越しに提供するだけでもソース開示義務」が発生し得る。商用SaaS/納品物に
  組み込むなら (a) Ultralytics Enterprise License を購入するか、(b) 排除する。
- **本リポジトリには既に排除経路がある**: 道路面モデルは U-Net / DeepLabV3+
  （segmentation_models_pytorch = MIT, PyTorch/torchvision = BSD）で完結する。
  `road_seg/segmenter.py` にも「smp の U-Net / DeepLabV3+ (MIT) は安全」と明記済み。
- **方針**: 商用ビルドでは幅推定を road_seg（MIT/BSD系）のみに限定し、Ultralytics 依存
  （SV YOLO・train_yolo.py）は開発検証用として隔離を維持。`models_yolo/` の重みを
  製品パッケージに同梱しない。

### 2.2 Google Street View 由来データの保存は ToS 抵触リスク
- Google Maps Platform の利用規約は、Street View 画像から**派生データを抽出・保存・
  データベース化することを許可していない**（画像の表示利用が原則）。
- SV+YOLO で推定した幅 (width_ai) を**永続保存して判定DBに使う運用は商用では不可**と
  みなすべき。**方針**: SVスキャンはセッション内の参考表示に限定し、保存される幅ソースは
  fgd_edge / 航空写真(GSI) / 手動上書き / OSM に限定する。
  （`runRealPerceptionFusion` の自動適用は商用ビルドでは航空写真ソースへ差し替える）

### 2.3 その他のデータ源（対応は「出典表示」中心・いずれも商用可）
- **地理院タイル/航空写真・基盤地図情報(FGD)・DEM5A**: 出典明記で商用利用可
  （測量法上の扱いに注意。Webサイト/成果物に「出典: 国土地理院」を表示）。
- **PLATEAU 3D Tiles**: 政府標準利用規約（CC-BY 4.0互換）。出典表示で商用可。
- **OpenStreetMap**: **ODbL 1.0**。(1) クレジット表示必須（© OpenStreetMap contributors）、
  (2) OSMデータを他ソースと混ぜて作った「派生データベース」（例: width_ai を焼き込んだ
  道路DB）を第三者へ**再配布**する場合は ODbL での公開義務（share-alike）が発生し得る。
  自社内利用・判定結果（Produced Work）の提供は問題なし。**道路DBそのものを顧客へ
  納品する形態を取るなら法務確認が必要**。
- アプリ画面に出典クレジット行（国土地理院 / OSM contributors / PLATEAU）を常設すること。

## 3. 特許・アルゴリズム面
本シミュレータが使う手法は全て数十年来の教科書的手法であり、特定社の専有実装を
複製していない: キネマティック自転車モデル、Pure Pursuit 追従（CMU, 1990年代）、
横加速度上限によるカーブ速度 v=√(a·R)、垂線断面の中央値による幅計測（写真測量の定石）、
固定タイムステップ決定論シミュレーション。既知の係争特許は存在しない領域だが、
本ドキュメントは法的助言ではない（製品化時に弁理士の FTO 確認を推奨）。
